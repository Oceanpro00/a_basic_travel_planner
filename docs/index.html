Travel Planner Project

A modular, browser-based system for planning and sharing interactive travel maps

Created as a passion project while learning web development and data science — built using a mix of custom logic and AI-assisted coding workflows.

Project Versions

Each version serves different use cases, from static map display to interactive trip planning:

v0.1 – Static Map Viewer

Display Only

Read-only map display where trip data is pre-configured. Perfect for showcasing completed itineraries.

✓ Best for:

  • Developers who want maximum control and customization
  • Projects requiring multiple trip variations (easy to swap data files)
  • Lightweight embedding in existing web applications

✗ Limitations:

  • Requires coding knowledge to set up and modify
  • Manual process for adding or changing locations

v0.2 – Interactive Planner

Interactive

Full-featured browser-based planner with drag-and-drop interface. Build custom trips and preview changes in real-time.

✓ Best for:

  • Users who want to build trip maps but can't/won't write code
  • Learning how trip data structures work through a visual interface
  • Situations where you want immediate visual feedback while building
  • Importing AI-generated trip data or copying code from other sources

✗ Limitations:

  • Tedious one-by-one input process (slower than bulk coding)
  • Manual location entry only (no autocomplete or search)

About This Project

This travel planner represents one of my early full-scale web development projects — part of my self-guided learning journey into frontend development, product design, and data visualization.

The project was built using a collaborative approach with generative AI tools (ChatGPT, Claude) to help me experiment, debug, and push beyond my current comfort zone. While I have other projects where I worked fully independently, this one specifically showcases my ability to build and adapt with AI collaboration in mind.

The focus has been on creating a modular, privacy-first alternative to commercial travel planning tools — one that works entirely in the browser without requiring accounts, databases, or data collection. While we may introduce optional convenience features in the future (autocomplete, cloud sync, AI integration), all core functionalities will remain free and accessible. Any premium features will be ad-free and only added to enhance the experience, never to lock away basic functionality.

⚡ AI-Powered Trip Creation

🚀 Why Use AI for Trip Data?

  • Much faster than the v0.2 UI for large trips (20+ locations)
  • More comprehensive - AI suggests places you might miss
  • Perfect coordinates - AI is surprisingly accurate with lat/lng
  • Consistent formatting - follows the exact API structure

📋 Step-by-Step Process

  1. Get the Resources:
    📖 Download Cheat Sheet 📝 View Example tripData.js
  2. Open ChatGPT (free version works perfectly)
  3. Upload both files or copy-paste their contents
  4. Use this prompt template:

💬 Copy These Prompts

Option A: Complete Trip Generation
Create a tripData.js file for a [X-day trip to LOCATION].
Use the API functions from the uploaded cheat sheet.

Include:
- 5-8 path points (main cities/areas)
- 10-15 activities and must-dos
- 8-12 restaurants and cafes
- Hotel suggestions for each path point
- Real coordinates (lat/lng)
- Placeholder photo URLs (I'll replace these myself)

Follow the structure from the uploaded example file.
Option B: Build Around Your Interests
Create a tripData.js file for [LOCATION] focused on [YOUR INTERESTS].
Use the API functions from the uploaded cheat sheet.

I specifically want to include:
- [Specific place 1] - because [reason]
- [Specific place 2] - because [reason]
- [Activity type] experiences
- [Food preference] restaurants

Build a complete trip around these priorities, adding complementary locations.
Include real coordinates and placeholder photo URLs.
Follow the structure from the uploaded example.

⚡ Pro Tips

  • Be specific: "Focus on art museums and local food markets"
  • Include must-sees: "I definitely want to visit [specific place]"
  • Set the vibe: "Budget-friendly" or "luxury travel" or "family-friendly"
  • Photo warning: AI photo URLs are usually broken - replace them yourself
  • Iterate easily: Ask for adjustments like "Add more cafes in downtown area"

🎯 After AI Generation

  1. Copy the generated tripData.js code
  2. Download the empty template
  3. Replace the tripData.js file with your generated code
  4. Add real photo URLs (this step is crucial!)
  5. Open index.html locally or deploy to any hosting service
